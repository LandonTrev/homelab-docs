# Session - 2026-07-13 - VLAN 10 build and isolation test

## Goal

Create an isolated lab network (VLAN 10) so the domain controller can eventually own DHCP and DNS without conflicting with the home router.

## Physical changes

- Moved the router ethernet cord from the MS-01 to the MikroTik CRS310 (ether1)
- Added a short ethernet cord from the switch (etherX) to the MS-01 management port
- New chain: router - switch - server (matches the planned apartment layout)
- Thick 10GbE DAC remains: switch sfp-sfpplus1 - MS-01 X710 port

## Switch configuration (MikroTik, WinBox)

- Logged in via MAC address in WinBox Neighbors, factory password from device label, new admin password set and stored securely
- Bridge - VLANs: added VLAN ID 10, tagged on bridge and sfp-sfpplus1, untagged none
  - Tagged sfp-sfpplus1: allows VLAN 10 frames to travel the DAC trunk, labels kept on
  - Tagged bridge: lets the switch CPU join VLAN 10 later for inter-VLAN routing
- Bridge - VLAN Filtering: enabled after verifying the table
- Untagged (default VLAN) traffic untouched, so management access survived the change

## ESXi configuration

- Physical NIC mapping confirmed: vmnic1 = 10GbE (DAC, link up), vmnic3 = 2.5GbE (management cord, link up), vmnic0 and vmnic2 unused
- vSwitch0 (uplink vmnic3) unchanged: Management Network and VM Network (DC01)
- Created vSwitch-Lab with uplink vmnic1
- Created port group Lab-VLAN10 on vSwitch-Lab, VLAN ID 10
- Ignored single-uplink redundancy warning (acceptable for lab)

## Isolation test

- Added a temporary second vNIC to DC01 on Lab-VLAN10
- Configured static IP 10.10.10.5/24, no gateway, no DNS
- ping 10.10.10.5: replies (adapter and tagging chain working)
- ping -S 10.10.10.5 to home router 10.0.0.1: transmit failed (no route off VLAN 10 - isolation confirmed)
- Removed the temporary vNIC afterward to avoid multi-homed DC issues

## State at end of session

- VLAN 10 exists and is enforced on the switch
- Lab-VLAN10 port group ready on the 10GbE path
- VLAN 10 is intentionally an island: no gateway, no internet
- DC01 unchanged, still on VM Network with its original static IP
